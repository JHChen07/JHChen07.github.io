1. DOCTYPE 和浏览器渲染模式

一个文档类型标记是一种标准通用标记语言的文档类型声明，目的是告知通用标记语言解析器，应该是用什么文档类型定义（DTD）来解析文档。DOCTYPE 还会对浏览器的渲染模式产生影响，不同的渲染模式会影响到浏览器对于 CSS 代码甚至是 JavaScript 脚本的解析，尤其是在 IE 系列浏览器中。因此，由 DOCTYPE 所决定的 HTML 页面的渲染模式至关重要。

1.1. 浏览器解析 HTML 方式

  1. 非怪异（标准）模式：页面按照 HTML 和 CSS 的定义渲染
  2. 怪异模式：尝试模拟旧浏览器的行为
  3. 部分怪异（近乎标准）模式：除了实施一种表单元格尺寸的怪异行为，其他符合标准

注：一个不含任何 DOCTYPE 的网页将会以怪异模式渲染。

2. meta 标签及其属性

只存在于 head 元素内部的 <meta> 元素可提供有关页面的元信息，<meta> 标签位于文档的头部，不包含内容。它的属性定义了与文档有关的名称/值对。在 HTML 中，<meta> 标签没有结束标签，而其在 XHTML 中必须被正确地关闭。
meta 标签共有两个属性，name 和 http-equiv。

2.1. name 属性

name 属性主要用于描述网页，与之对应的属性值为 content，content 中的内容主要是便于搜索引擎机器人查找信息和分类信息用的。
meta标签的name属性语法格式是： 

<meta name="参数"content="具体的参数值">。

各个参数及其作用如下表：

参数 参数作用
keywords 告知搜索引擎网页的关键字
description 网站内容描述
robots 机器人向导，告知其页面是否需要索引
author 标注网站的作者
generator 说明网页制作的软件
revised
others

2.2. http-equiv

http-equiv 属性为名称/值对提供了名称。并指示服务器先在要传送的 MIME 文档头部包含名称/值对。至少要包含 content-type:text/html 。与之对应的属性值为 content，例如： 

<meta http-equiv="charset" content="iso-8859-1">
<meta http-equiv="expires" content="31 Dec 2008">

发送给浏览器的文档头部包含： 

content-type: text/html
charset:iso-8859-1
expires:31 Dec 2008

各项参数及其作用如下表：

参数 参数作用
content-type 显示字符集的设定
expire 设定网页的到期时间，过期之后需要重新传输
pragma(cache 模式) 禁止浏览器从本地计算机的缓存中访问页面内容
refresh 自动刷新并指向新页面
set-cookie 设置 cookie，如果网页过期，那么存盘的 cookie 将被删除，必须使用 GMT的时间格式
window-target 强制页面在当前西黄口以独立页面显示
content-language 显示语言的设定
cache-control 指定请求和相应遵循的缓存机制

meta 标签可以便于搜索引擎排序，提高搜索引擎网站权重排名，定义页面使用的语言，自动刷新并指向新的页面，实现网页转换时的动态效果，控制页面缓冲，网页定级评价，控制网页显示的窗口等。

3. DOM 结构

DOM：Document Object Module，文档对象模型。

3.1. W3C DOM 标准

  1. 核心 DOM - 针对任何结构化文档的标准模型
  2. XML DOM - 针对 XML 文档的标准模型
  3. HTML DOM - 针对 HTML 文档的标准模型

3.2. DOM 节点

根据 W3C 的 HTML DOM 标准，HTML 文档中的所有内容都是节点
文档节点、元素节点、属性节点、文本节点、注释节点等。

3.3. HTML DOM节点树

HTML 文本会被解析为 DOM 树，树中的所有节点均可通过 JavaScript 进行访问。所有HTML 元素（节点）均可被修改，也可以创建或删除节点。

3.4. 节点的关系

父、子、同胞等术语用于描述这些关系。节点树中，顶端节点被称为根；每个节点都有父节点，除了根；一个节点可以拥有任意数量的子，同胞拥有相同的根。

4. 元素分类

在 CSS 中，HTML 中的标签元素可以分为三种不同的类型：

4.1. 块级元素（区块元素）

display: block;

div、p、h1、…h6、ol、ul、dl、dt、dd、table、address、blockquote、pre、form、hr、article、aside、footer、header、hgroup、main、nav、section等。其默认特点为：

  1. 每个块级元素都从新的一行开始，包括其后的元素。
  2. 元素的高度、宽度、行高以及顶和底边距都可以设置。
  3. 元素宽度在不设置的情况下，是它父元素的 100%， 除非设定。

4.2. 内联元素：

display: inline;

span、a、label、strong、em、i、q、var、cite、code 等。默认特性：

  1. 与其它元素在同一行。
  2. 元素的高度、宽度、行高以及顶和底边距都不可以设置。
  3. 元素的宽度就是它包含的文字或图片的宽度，不可更改。

4.3. 内联块级元素：

display: inline-block;

img、input。默认特性：

  1. 与其它元素在同一行。
  2. 元素的高度、宽度、行高以及顶和底边距都可以设置。

5. 盒模型

每个元素都是一个矩形盒子，盒模型是元素大小的呈现方式。每个盒子有四个边：外边距边，边框边，内填充边与内容边。

盒子模型分为两种，一种是 W3C 标准盒模型，一种是 IE 盒模型。

  1. content-box （标准盒模型）
    width = content 宽度
    height = content 高度
  2. border-box （IE 盒模型）
    width = border + padding + content 宽度
    height = border + padding + content 高度

以上两者可以通过 box-sizing 属性自由转换，默认为 content-box。

5.1. JavaScript 如何设置获取盒模型对应的宽和高

  1. dom.style.width/height 只能取到行内样式的宽和高，style 标签中和 link 外链的样式取不到。
  2. dom.currentStyle.width/height 取到的是最终渲染后的宽和高，只有 IE 支持此属性。
  3. window.getComputedStyle(dom).width/height 同（2）但是多浏览器支持，IE9以上支持。
  4. dom.getBoundingClientRect().width/height 也是得到渲染后的宽和高，大多浏览器支持。IE9 以上支持，除此外还可以取到相对于视窗的上下左右的距离

5.2. 外边距重叠

当两个或更多个垂直边距相遇时，它们将形成一个外边距。这个外边距的高度等于两个发生叠加的外边距的高度中的较大者。但是只有普通文档流中的快款的垂直外边距才会发生外边距叠加。行内框、浮动框或绝对定位框之间的外边距不会叠加。
垂直外边距叠加有三种情况：

  1. 元素自身叠加。当元素没有内容、内边距、边框时，它的上下边距就相遇了，即会在垂直方向产生叠加。当添加上述任何一项，叠加就会取消。
  2. 相邻元素叠加。
  3. 包含（父子）元素叠加。包含元素的外边距隔着父元素的内边距和边框，当这两项都不存在时，父子元素垂直外边距相邻，产生叠加。添加任何一项就会取消叠加。

6. BFC

BFC（Block Formatting Context）：块格式化上下文。
BFC 决定了元素如何对其内容进行定位，以及与其他元素的关系和相互作用。当设计到可视化布局时，BFC提供了一个环境，HTML 元素在这个环境按照一定的规则进行布局。一个环境中的元素不会影响到其它环境中的布局。

6.1. BFC 原理（渲染规则）

  1. BFC 元素垂直方向的边距会发生重叠。属于不同 BFC 外边距不会发生重叠。
  2. BFC 的区域不会与浮动元素的布局重叠。
  3. BFC 元素是一个独立的容器，外面的元素不会影响里面的元素。反之亦然。
  4. 计算 BFC 高度时，浮动元素也会参与计算（清除浮动）

6.2. 如何创建 BFC

  1. 根元素或包含根元素的元素
  2. 浮动元素（元素的 float 不是 none）
  3. 绝对定位元素（元素的 position 为 absolute 或 fixed）
  4. 行内块元素（元素的 display 为 inline-block）
  5. 表格单元格（元素的 display 为 table-cell，HTML 表格单元格默认为该值）
  6. 表格标题（元素的 display 为 table-caption，HTML 表格标题默认为该值）
  7. 匿名表格单元格元素（元素的 display 为 table、table-row、 table-row-group、table-header-group、table-footer-group（分别是HTML table、row、tbody、thead、tfoot 的默认属性）或 inline-table）
  8. overflow 值不为 visible 的块元素
  9. display 值为 flow-root` 的元素
  10. contain 值为 layout、content 或 strict 的元素
  11. 弹性元素（display为 flex 或 inline-flex 元素的直接子元素）
  12. 网格元素（display 为 grid 或 inline-grid 元素的直接子元素）
  13. 多列容器（元素的 column-count 或 column-width 不为 auto，包括 column-count 为 1）
    column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更，Chrome bug）

注：块格式化上下文包含创建它的元素内部的所有内容。浮动定位和清除浮动时只会应用于同一个 BFC 内的元素。浮动不会影响其他 BFC 中元素的布局，而清除浮动只能清除同一 BFC 中在它前面的元素的浮动。外边距折叠（Margin Collapsing）也只会发生在属于同一 BFC 的块级元素之间。

7. CSS普通流（文档流）

元素按照其在 HTML 中的位置顺序决定排布的过程。

7.1. 调节普通流元素位置

一般使用 margin 是用来隔开元素与元素的间距；padding 是用来隔开元素与内容的间隔。margin 用于布局分开元素使元素与元素互不相干；padding 用于元素与内容之间的间隔，让内容（文字）与（包裹）元素之间有一段“距离”。
注：只要不是 float 和绝对定位方式布局的，都在普通流里面。

link 标签和 @import 的区别

参考网上博客的总结，两者都是外部引用 CSS 的方式，但是存在一定区别：

  1. link 除了引用样式文件，还可以引用图片等资源文件，而 @import 只引用样式文件；
  2. link 引用 CSS 时，在页面载入时同时加载；@import 需要网页完全载入以后加载；
  3. link 是 XHTML 标签，无兼容问题，@import 是在 CSS2.1 时提出的，低版本的浏览器不支持；
  4. link 支持使用 JavaScript 控制 DOM 去改变样式，而 @import 不支持。

8. 变量对象

8.1. JavaScript 的基本类型

  1. Number
  2. String
  3. Boolean
  4. Undefined
  5. Null
  6. Object

五种基本类型保存在内存的栈中，值固定，复制其变量时会创建这个值得一个副本。使用 typeof 区分。前五中没有原型没有构造函数。

8.2. 引用类型

引用类型的值是对象，保存在堆中。引用类型的变量实际上是一个指针，他保存在栈中，指向堆内存中的对象。复制引用类型变量实际是复制该指针，所以他们都指向同一个对象，用instanceof确定哪一个值是哪种引用类型。

8.3. 内置对象、原生对象及宿主对象

内置对象和元素对象是被 ECMAScript 规范定义和实现的，两者之间的差异微不足道。所有 ECMAScript 实现的对象都是原生对象（其中一些是内置对象、一些在程序执行的时候创建，例如用户自定义对象）。内置对象是原生对象的一个子集、是在程序开始之前内置到 ECMAScript 里的（例如，parseInt, Match等）。所有的宿主对象是由宿主环境提供的，通常是浏览器，并可能包括如 window、alert 等。
注意，宿主对象可能是 ES 自身实现的，完全符合规范的语义。从这点来说，他们能称为“原生宿主”对象（尽快很理论），不过规范没有定义“原生宿主”对象的概念。

8.4. 特殊包装类

  1. Boolean: 布尔对象
  2. String: 字符串对象
  3. Number：数字对象

这些对象的创建，是通过相应的内置构造器创建，并且包含原生值作为其内部属性，这些对象可以转换成原始值，反之亦然。 

    var c = new Boolean(true);
    var d = new String('test');
    var e = new Number(10);

    // 转换成原始值
    // 使用不带 new 关键字的函数
    с = Boolean(c);
    d = String(d);
    e = Number(e);

    // 重新转换成对象
    с = Object(c);
    d = Object(d);
    e = Object(e);

8.5. typeof返回值

实际是用来检测一个对象是否已经定义或者是否已经赋值。

Value Class Type
“foo” String string
1.2 Number number
true Boolean boolean
undefined Undefined undefined
new Function(“”) Function function
new String(“foo”) String object
new Number(1.2) Number object
new Boolean(true) Boolean object
new Date() Date object
new Error() Error object
[1,2,3] Array object
new Array(1, 2, 3) Array object
/abc/g RegExp object (function in Nitro/V8)
new RegExp(“meow”) RegExp object (function in Nitro/V8)
{} Object object
new Object() Object object

返回的类型只有 6 种：

  1. string
  2. number
  3. boolean
  4. undefined
  5. object
  6. function

8.6. instanceof 操作符

instanceof 操作符用来比较两个操作数的构造函数。只有在比较自定义的对象时才有意义。如果用来比较内置类型，将会和 typeof 操作符 一样用处不大。

8.7. Object.prototype.toString 获取对象类型

检测一个对象的类型，强烈推荐使用 Object.prototype.toString 方法； 因为这是唯一一个可依赖的方式。typeof 的一些返回值在标准文档中并未定义， 因此不同的引擎实现可能不同。 我们使用Object.prototype.toString方法: 

Object.prototype.toString.call([])    // "[object Array]"
Object.prototype.toString.call({})    // "[object Object]"
Object.prototype.toString.call(2)    // "[object Number]"
