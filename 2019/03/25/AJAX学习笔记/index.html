AJAX学习笔记

发表于 2019-03-25 | 分类于 前端
字数统计: 1.7k | 阅读时长 ≈ 6

此文是学习 AJAX 教程的笔记摘要。

定义

AJAX(Asynchronous JavaScript + XML), 能够向服务器请求额外的数据而无需卸载页面，是一种用于创建动态网页的技术，通过在后台与服务器进行少量数据交换， AJAX 可以使网页实现异步更新。带来更好的用户体验。
AJAX 不是新的编程语言，而是一种使用现有标准的新方法。
AJAX 最大的优点是在不重新加载整个页面的情况下，可以与服务器交换数据并更新部分网页内容。
AJAX 不需要任何浏览器插件，但需要用户允许 JavaScript 在浏览器上执行。
工作原理(图源自网络)：

AJAX 基于现有的 Internet 标准，独立于浏览器和平台，并联合使用：

  1. XMLHttpRequest 对象 (异步的与服务器交换数据)
  2. JavaScript/DOM (信息显示/交互)
  3. CSS (给数据定义样式)
  4. XML (作为转换数据的格式)

XMLHttpRequest 对象

XMLHttpRequest 对象（简称 XHR），是 AJAX 技术的核心。在 XHR 出现之前， AJAX 式的通信必须借助一些 hack 手段来实现，大多数是使用隐藏的框架或内置框架。XHR 为向服务器发送请求和解析服务器相应提供了流畅的接口。能够以异步方式从服务器取得更多信息（可以使用 XHR 对象取得新数据，然后再通过 DOM 将新数据插入到页面中）。
所有现代浏览器（IE7+、Firefox、Chrome、Safari 以及 Opera）均内建 XMLHttpRequest 对象。
创建 XMLHttpRequest 对象： 

variable = new XMLHttpRequest();

老版本的 Internet Explorer （IE5 和 IE6）使用 ActiveX 对象： 

variable = new ActiveXObject("Microsoft.XMLHTTP");

为了应对所有的现代浏览器，包括 IE5 和 IE6，请检查浏览器是否支持 XMLHttpRequest 对象。如果支持，则创建 XMLHttpRequest 对象。如果不支持，则创建 ActiveXObject。

使用 XMLHttpRequest 对象

向服务器发送请求

open() 和 send() 方法：

方法 描述
open(method,url,async) 规定请求的类型、URL 以及是否异步处理请求，
并不会真正发送请求，知识启动一个请求以备发送。
method：请求的类型；GET 或 POST
url：文件在服务器上的位置
async：true（异步）或 false（同步）
send(null/string) 将请求发送到服务器。
string：仅用于 POST 请求
null: 不需要发送数据必须传入 null

XMLHttpRequest 对象属性

responseText：获得 string 形式的响应数据。
responseXML：获得 XML 形式的响应数据。
status：HTTP 响应的状态
statusText：HTTP 状态的说明
发送请求以及处理响应示例： 

var xhr = new XMLHttpRequest();
xhr.open('get', 'example.txt', false);
xhr.send(null);
// 304 表示请求的资源并没有被修改，可以直接使用浏览器缓存的数据
if((xhr.status >= 200 && xhr.status <300) || xhr.status == 304) {
    alert(xhr.responseText);
} else {
    alert('request was unsuccessful: ' + xhr.status);
}

onreadystatechange 事件

当请求被发送到服务器时，我们需要执行一些基于响应的任务。
每当 readyState 改变时，就会触发 onreadystatechange 事件。readyState 属性存有 XMLHttpRequest 的状态信息。
下面是 XMLHttpRequest 对象的三个重要的属性：

属性 描述
onreadystatechange 存储函数（或函数名），每当 readyState 属性改变时，就会调用该函数。
readyState 存有 XMLHttpRequest 的状态。从 0 到 4 发生变化。
0: 请求未初始化，尚未调用 open() 方法
1: 启动，服务器连接已建立，已经调用 open()， 尚未调用 send()
2: 发送，已经调用 send()，尚未接收到响应
3: 接收，已经接收到部分响应数据
4: 完成，请求已完成，且响应已就绪
status 示例：
200: “OK”
404: 未找到页面

一旦 readyState 属性的值产生变化，都会触发一次 onreadystatechange 事件。必须在 open() 之前指定 onreadystatechange 事件处理程序才能确保跨浏览器兼容性： 

var xhr = new XMLHttpRequest();
// 采用 DOM 0 级方法为 XHR 对象添加事件处理程序，并且没有传递 event 对象，考虑了浏览器的兼容性
xhr.onreadystatechange = function() {
    if(xhr.readyState == 4) {
        if((xhr.status >= 200 && xhr.status <300) || xhr.status == 304) {
            alert(xhr.responseText);
        } else {
            alert('request was unsuccessful: ' + xhr.status);
        }
    }
};
xhr.open('get', 'example.txt', true);
xhr.send(null);

另外，在接收到响应之前还可以调用 abort() 方法来取消异步请求： 

xhr.abort();

调用该方法后，XHR 对象会停止触发事件，而且也不再允许访问任何与响应有关的对象属性。在终止请求后，还应该对 XHR 对象进行解引用，也不建议重用 XHR 对象。

XMLHttpRequest 2级

XMLHttpRequest 2 级是 XHR 对象的发展，但并非所有浏览器都完整实现了 XMLHttpRequest 2 级规范。

FormData

FormData 用于序列化表单以及创建与表单格式相同的数据（用于通过 XHR 传输）。使用 FormData 的方便之处在于不必明确地在 XHR 对象上设置请求头部。XHR 对象能够识别传入的数据类型是 FormData 的实例，并配置适当的头部信息。

示例

新建实例： 

var data = new FormData();
// 可以使用 append() 方法添加任意多个键值对
data.append('name', 'David');
// 也可以向 FormData 的构造函数中传入表单元素，也可以用表单元素的数据预先向其传入键值对
// var data = new FormData(document.forms[0]);

完整示例： 

var xhr = new XMLHttpRequest();
// 采用 DOM 0 级方法为 XHR 对象添加事件处理程序，并且没有传递 event 对象，考虑了浏览器的兼容性
    xhr.onreadystatechange = function() {
        if(xhr.readyState == 4) {
            if((xhr.status >= 200 && xhr.status <300) || xhr.status == 304) {
                alert(xhr.responseText);
            } else {
                alert('request was unsuccessful: ' + xhr.status);
            }
        }
    };
xhr.open('post', 'example.php', true);
var form = document.getElementById('info');
xhr.send(new FormData(form));

超时设定

IE8 为 XHR 对象添加了一个 timeout 属性，表示请求在等待响应多少毫秒之后就终止。在设定的事件之后浏览器还没有接收到响应，就会触发 timeout 书剑，进而会调用 ontimeout 事件处理程序。超时终止后，可能 readyState 会发生改变，浏览器调用 onreadystatechange 事件处理程序，但是在请求终止后再访问 state 属性，就会导致错误。为避免浏览器报告错误，可以将此部分用 try-catch 块处理。 

var xhr = new XMLHttpRequest();
xhr.onreadystatechange = function() {
    if(xhr.readyState == 4) {
        try {
            if((xhr.status >= 200 && xhr.status <300) || xhr.status == 304) {
                alert(xhr.responseText);
            } else {
                alert('request was unsuccessful: ' + xhr.status);
            }
        } catch(ex) {
            // 处理错误
        }

    }
};
xhr.open('get', 'example.txt', true);
xhr.timeout = 1000;
xhr.ontimeout = function() {
    // 处理错误
};
xhr.send(null);

overrideMimeType() 方法

用于重写 XHR 响应的 MIME(Multiplurpose Internet Mail Extensions，多用途互联网邮件扩展) 类型： 

var xhr = new XMLHttpRequest();
xhr.open('get', 'example.txt', true);
xhr.overrideMimeType('text/xml');
xhr.send(null);
