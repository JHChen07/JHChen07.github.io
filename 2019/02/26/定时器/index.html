定时器

JavaScript提供了一些原生的方法去延时去执行一段代码。指定的时间间隔表示何时将定时器的代码添加到队列，而不是何时执行代码。定时器有四种形式。

setTimeout

设置一个定时器，在定时器到期后执行一次函数或代码段。 

var timeoutId = window.setTimeout(func[, delay, param1, param2, ...]);
var timeoutId = window.setTimeout(code[, delay]);

参数说明：

  1. timeoutId: 定时器 ID
  2. func: 延迟后执行的函数
  3. code: 延迟后执行的代码字符串，不推荐使用原理类似 eval()
  4. delay: 延迟的时间（单位：毫秒），默认值为 0
  5. param1,param2: 向延迟函数传递而外的参数，IE9 以上支持

setInterval

以固定的时间间隔重复调用一个函数或者代码段 

var intervalId = window.setInterval(func, delay[, param1, param2, ...]);
var intervalId = window.setInterval(code, delay);
  1. intervalId: 重复操作的 ID
  2. func: 延迟调用的函数
  3. code：代码段
  4. delay：延迟时间，没有默认值

当使用 setInterval 时， 仅当没有该定时器的其他任何代码实例时，才将定时器的代码添加到队列中。这确保了定时器代码加入到队列中的最小时间间隔为指定时间间隔。
这种重复定时器的规则有两个问题：（1）某些时间间隔会被跳过；（2）多个定时器的代码执行之间的间隔可能会比预期的小。假设，onclick 事件处理程序使用 setInterval() 设置了一个 200ms 的间隔重复定时器。如果事件处理程序花了 200ms 多一点的事件完成，此时，将定时器添加到队列中，然后定时器代码也花了超过 200ms 的时间，重复定时器代码副本再次添加到队列中去。但此时，队列中已经有了另一个定时器代码的实例，这时就会出现跳过 605ms 添加的定时器代码并且连续执行 405ms 定时器代码的情况。如图所示。

为了避免 setInterval() 设置的重复定时器的这2个缺点，可以使用 setTimeout() 进行链式调用： 

setTimeout(function() {
    //处理中   
    setTimeout(arguments.calle, interval);
}, interval);

这段代码的作用是，每次函数执行的时候都会创建一个新的定时器。第二个 setTimeout() 调用使用了 arguments.calle 来获取对当前执行函数的引用，并为其设置一个新的定时器。因此，在前一个定时器代码执行完毕之前，不会向队列插入新的定时器代码，确保不会有任何缺失的间隔。而且，它可以保证，在下一次定时器代码执行之前，至少等待指定的间隔，避免了连续执行。这模式主要用于重复定时器。 

setTimeout(function() {
    var div = document.getElementById('myDiv');
    left = parseInt(div.style.left) + 5;
    div.style.left = left + 'px';
    if(left < 200) {
        setTimeout(arguments.calle, 50)
    })
}, 50);

setImmediate

在浏览器完全结束当前运行的操作后立即执行指定的函数（仅 IE10 和 node 0.10
+中有实现），类似 setTimeout(func, 0) 

var immediateId = setImmediate(func[, param1, param2, ...]);
var immediateId = setImmediate(func);
  1. immediateId：定时器ID
  2. func：回调

requestAnimationFrame

专门为实现高性能的帧动画而设计的API，但是不能指定延迟时间，而是根据浏览器的刷新频率而定（帧） 

var requestId = window.requestAnimationFrame(func);

注意

  1. setTimeout 有最小时间间隔限制，HTML5 标准为 4ms，小于 4ms 按照 4ms 处理，但是每个浏览器实现的最小间隔都不同
  2. 因为 JS 引擎只有一个线程，所以它将会强制异步事件排队执行
  3. 如果 setInterval 的回调执行时间长于指定的延迟，setInterval 将无间隔的一个接一个执行
  4. this 的指向问题可以通过 bind 函数、定义变量、箭头函数的方式来解决
